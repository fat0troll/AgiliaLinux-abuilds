About distro

What is AgiliaLinux?

AgiliaLinux is a completely non-commercial Linux distribution being developed by Russian community.
AgiliaLinux was started as a fork of another russian distro - MOPSLinux, which was discontinued in April 2010.
AgiliaLinux, as a fork of MOPSLinux, got it's roots from Slackware Linux, but nowadays it is completely independent from any other distro. In other words, saying «AgiliaLinux is a Slackware-based distribution» is not completely correct.
AgiliaLinux aims to attempt to combine principles of KISS and user-friendly, bleeding-edge and acceptable level of stability. Easy, Simple, Fresh. And, of cource, fast.
Our users are mostly Russian-speaking people, so we provide significant attention to Russian localization quality. But we provide multilingual system.

Current status

At this moment, everything is in very very active development, but, since we are all already using AgiliaLinux as our primary and the only OS in many different tasks and environments, we are trying to always keep it as stable as possible.

It means that it is ready to use in most cases, but it may require attention at some updates. Also, nobody can guarantee that some update would not suddenly break your system completely or eat your cat, altrough we are trying to avoid this situations (by the way, there was none of them - only non-critical breakages).

There is no oficial stable release, but we do release "stable" snapshots. It is done this way just because we are still in progress of implementing everything we want in the base system. Altrough we are trying to make any updates easy and seamless, it is theoretically possible that at some moment you will have to reinstall your system to update next. But we hope this won't happen :)

Very tiny FAQ about AgiliaLinux

Q: When you will release an official stable version?
A: Never. Yes, it's only joke :) Right after we decide that there is nothing left to change in base system, and after some reasonable time spent to tests. No one in galaxy can say exactly when it will happen, but this time is closer and closer… Some people says that it will be in 31th December, 2010, but what can be…
Q: I'm a complete noob in Linux. Should I try AgiliaLinux or should I look for something else?
A: If you want to spend some time and understand how it works, and are not afraid of green 'Matrix' console - yes, you should definitely try it. If you want everything to be configured via nasty GUI control center, here is no one (still), and you can fall into a panic if something will not work as expected.
Q: Do you have a LiveCD?
A: LiveCD planned in future. Today we have LiveDVD, which you will use for installing AgiliaLinux. It contains rich Openbox-session. Try it, and you'll like it :)
Q: What is a primary desktop environment in AgiliaLinux?
A: We provide KDE, GNOME and Openbox as most supported and tested environments, so all of them are primary. In addition, we provide LXDE, XFCE, and Fluxbox setup variants out of the box. There is a plan to support E17 installation. All these environments are ready to use out of the box.

Release notes

157 days we spend before Beta 1, it's not accidentally: we rebuild almost half of our packages, and adds many of new stuff. All remember this summer in Russia, it was very fry, as nuclear, it was a reason of very long interval between Snapshot 4 and Beta 1, and we dedicate this Beta for this summer. So, meet AgiliaLinux 8.0 Beta 1 "Nuclear Summer"!

Most important changes

  • New GTK+-theme Prudence,which was maded by Klogg on Murrine and Equinox engines, now in default KDE and GNOME.
  • KDE: update to latest 4.5.2, with latest features of fresh desktop. Dropped ktheme, old KDE theme, which starts from MOPSLinux, because it's very buggy in new KDE (or KDE is very buggy…). Oxygen is new Qt default theme.
  • GNOME: ready to use out-of-the-box. Yeah, sure. Updated to 2.32.0. Now GNOME has it's own settings and apps. Deleted garbage from Openbox-session, which was in Snapshot 4.
  • Linux Kernel: updated to 2.6.34.7. 2.6.36 will be soon, guys :)
  • X.org-server: hell's update. From 1.7 to 1.9 between 5 month. Now X.org setting stored not in xorg.conf, but in /etc/X11/xorg.conf.d/. With this update we remove HAL from GNOME and Openbox.
  • mpkg: faster, smarter, stronger! Now mpkg faster than never before, and work for fastering mpkg still in progress!
